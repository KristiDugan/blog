Getting Started with Your First Repository


Congratulations on starting your coding journey! You’re probably feeling excited and overwhelmed by all the new lingo that is rolling off of everyone’s tongues. Here are a few tips to help you ease into your new coding life.


One of the first accounts you’ll set up is a GitHub. GitHub is a repository, or a space in the cloud to save all your coding work. Think of it as Google Drive for your coding projects. Hopefully you’re thinking, “Hmmm, I can usually collaborate with others when using cloud computing…” You’re right! GitHub lets you collaborate with others, just like other cloud computing tools.


Next, you’ll want to learn about Git. But wait, didn’t I just set one up? Though, similarly named, they are very different. Git is a version control system. GitHub and Git work together to store and track changes of your coding work. Think back to the days before cloud computing, and saving different versions of your work: Resume v1, Resume v2, etc. (admittedly, I still do this). That’s what you’re doing with Git- committing changes to your coding files, and leaving yourself (or your collaborators) a note about what you did.


You’ll be using Git in a terminal. A terminal is text-based interface where you can interact with your files. When working in the terminal, you are always in a directory (or folder) somewhere on your computer. You can tell which directory you’re in by looking at the path.
